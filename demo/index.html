BridgeChat v1.0

An embeddable WebRTC chat client, which bridges the communication gap between old and new technologies.
BridgeChat WebRTC Demos

Custom Server Demo

The Custom Server Demo pairs the BridgeChat client with the included Java-based WebSocket server to provide multi-peer signaling.

Using the Custom Server implementation provides the following benefits:

  • Easily embedded into existing websites. Simply place a single div with the id 'bridgechat-container' and the client's UI Manager will do the rest!
  • Provides a fully functional signaling channel. No need to email or IM links to those you wish to chat with!
  • Support for multiple WebRTC connections (currently up to 3!)
  • Custom server can be modified in the future to provide advanced features such as call queuing for call center operations.
  • Easy installation. Just upload the distributable jar file to your web server and run it!

XMPP Demo

The XMPP Demo connects to the public Jabber.org server and uses the MUC protocol to create and join chat rooms, which are used as the signaling channel.

Using the included WebSocket to Socket proxy, this option provides multiple benefits of its' own, such as:

  • Shares same, embeddable Javascript client as the Custom version.
  • Uses existing XMPP servers as signaling channel.
  • Provides authentication options for extra security when user identity is important. Currently supports PLAIN authentication, but plans are in the works to add more types.
  • Uses XMPP's Multi-User-Chat protocol to handle session events between users.
  • Can connect directly (without use of the proxy) to XMPP servers which support WebSocket connections.
